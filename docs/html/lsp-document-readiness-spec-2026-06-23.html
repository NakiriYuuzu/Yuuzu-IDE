LSP Document Readiness Lifecycle

2026-06-23 · Review artifact · Source spec: docs/superpowers/specs/2026-06-23-lsp-document-readiness-design.md

核心設計: 不預設啟動所有 language servers，而是建立 document-first readiness contract。任何 editor LSP action 執行前，都先確保目前文件已啟動 server、initialized、didOpen/didChange 到最新內容。

問題定義

Current狀態語義混淆

Stopped 同時被用來表示「偵測到但未啟動」和「曾啟動後停止」。使用者會以為 LSP 壞了或應該手動啟動。

CurrentAction 前置條件分散

gotoDefinition、references、rename、code actions 直接送 request，沒有共同的 document readiness 入口。

Constraint不能暴力全開

workspace 開啟就啟動所有 LSP 會拖慢大型 repo，也仍可能留下 stale content 問題。

Target使用時保證可用

只要使用者對某個支援檔案觸發 LSP 行為，系統就自動準備該文件，再執行 request。

Lifecycle Diagram

Contract

Backend command

lsp_ensure_document({
  workspaceId,
  workspaceRoot,
  path,
  content,
  version
})

Detect language, spawn if needed, initialize, send didOpen/didChange, return document readiness and server status.

Frontend helper

await ensureLspDocumentReady(tab.id)
await requestLanguageDefinition(...)

Controller dedupes concurrent ensures per workspace/path and patches LSP UI state from the result.

State Split

LayerStatesPurpose
Server lifecycleIdle, Starting, Running, MissingCommand, Error, StoppedProcess lifecycle and Settings UI.
Document readinessUnsupported, Syncing, Ready, Stale, MissingCommand, ErrorWhether a specific file can safely receive LSP requests.

Verification Plan

Rust

  • Detected unopened language returns Idle.
  • Unsupported file returns Unsupported and does not spawn.
  • Idle + supported file ensures to Ready.
  • Changed content sends didChange.
  • Missing command returns MissingCommand with command name.

Frontend

  • Definition/references/rename/code actions call ensure first.
  • Context menu enables supported file actions even when server is Idle.
  • MissingCommand produces actionable toast.
  • Language Settings renders Idle and Start button.
Suggested gates: bun test src/features/language src/v2, bunx tsc --noEmit, cargo test --manifest-path src-tauri/Cargo.toml lsp, packaged debug app smoke.

Runtime Smoke

  1. Build debug app with updater artifacts disabled for local smoke.
  2. Fresh launch packaged app.
  3. Open a TypeScript file.
  4. Invoke Go to Definition or Code Actions without visiting Language Settings first.
  5. Confirm server transitions Idle/Starting to Running or MissingCommand with clear message.
