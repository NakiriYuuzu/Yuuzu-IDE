偏差詳情（5 項，皆為文件/慣例層級）

中 1. Windows 驗證缺口從未被記錄

產品定位是「Windows-capable」，tech-stack.md 也有 WebView2 發佈政策，但 Node 0–9 的所有驗證（DMG、.app、量測）都在 macOS 上完成，progress.md 全文沒有任何「Windows」字眼——沒有把「Windows 驗證不可行」列為 blocker 或 residual risk。

goal.md 要求：「If blocked by … unavailable Windows verification … stop and report the exact blocker」。實務上在 macOS 主機連續推進是合理選擇，但至少應在各 node 的 residual risks 記錄此限制。

建議：在 progress.md 補一段「Windows verification deferred」的全域 residual risk，並在 Node 13（packaging）前安排 Windows 驗證管道。

低 2. roadmap.md 的 Node 4、6、7 區塊缺 Status 行

Node 0/1/2/3/5/8/9 的區塊都有「**Status:** completed and passed; … results are recorded in …」，但 Node 4（roadmap.md:371）、Node 6（:428）、Node 7（:458）沒有。雖然「Current Priority」段與 progress.md 都記錄了完成，且三份 results 文件（node-4-git-results.md 等）都存在，但 goal.md 的 Done When 要求「roadmap.md reflects completed nodes accurately」——文件內部不一致。

建議：補上三行 Status 註記即可，5 分鐘工作量。

低 3. commit 訊息不符使用者全域 git 慣例

190 筆 commits 全部為 type: subject 簡式格式，沒有 scope、沒有 emoji、沒有 Co-Authored-By: Yuuzu <yuuzu@yuuzu.net> trailer（全域 ~/.claude/rules/git/commit.md 要求每筆 commit 以此結尾）。

另外 goal.md 規定「未經明確授權不得 commit」——repo 有 190 筆 commits，前提是 Goal run 取得了 commit 授權；此點無法從 repo 內部驗證，僅你本人能確認當初是否授權。

建議：若後續 node 繼續執行，在 Goal run 指令中明確帶入 commit 格式要求。

低 4. TDD 的 RED 階段缺失敗輸出原文

各 node results 記錄了 RED/GREEN 命令與週期描述（例如 Node 4 逐 task 描述 RED/GREEN cycles、Node 9 逐 task 列出命令），但未保存 RED 階段觀察到的失敗輸出原文。goal.md 要求「the failure must be observed for the expected reason」——目前的記錄只能間接推斷，無法事後稽核失敗原因是否符合預期。

建議：後續 node 在 results 中為每個 task 保留一段 RED 失敗訊息摘錄（一兩行即可）。

資訊 5. goal.md 引用的 AGENTS.md 不存在

goal.md 寫「Existing conventions: follow AGENTS.md and local project docs」，但 repo 根目錄沒有 AGENTS.md（也沒有 CLAUDE.md）。因為條文寫的是「existing」，檔案不存在即無約束力，不算違規；但 goal.md 引用了一個不存在的檔案，建議建立 AGENTS.md 或從 goal.md 移除該引用。
