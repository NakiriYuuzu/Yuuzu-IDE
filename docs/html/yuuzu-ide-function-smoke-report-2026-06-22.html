FIXED Git Graph 內容因 Git watcher events 反覆閃爍

重現：重新開啟目前 repo 的 Git Graph 後，commit detail 區塊會反覆回到 Loading commit detail…，看起來像整個內容一直閃。

根因：reloadGit() 每次會重設 gitSel 與 gitDetail；而 real watcher path 在 .git/* 內部事件上仍無條件 scheduleGitReload()，導致 Git Graph detail 被反覆清空再載入。

修復：controller 將 watcher absolute path 轉成 workspace-relative path，並套用既有 shouldRefreshGitAfterFileEvent()，忽略 .git/* 內部事件但保留一般檔案變更的 Git reload。

驗證：folder-expand.test.ts 新增 .git/index.lock watcher event regression test，先紅後綠；新 debug app 中 Git Graph detail 在 .git/codex-watch-probe.* 事件後維持 changed files/message，未回到 loading。

FIXED CodeMirror 語法色彩未接上 dark/light syntax theme token

現象：切換 light theme 後 editor 容器、gutter 與背景會變亮；但 CodeMirror token 高亮仍來自 defaultHighlightStyle，沒有使用 src/v2/yuzu.css 內定義的 --yz-syntax-* dark/light 變數。

修復：新增 Yuuzu HighlightStyle，token color 使用 --yz-syntax-* CSS variables。

驗證：editor-surface.test.ts 檢查 highlight specs 包含 Yuuzu syntax variables。

FIXED Dark theme 下 CodeMirror 輸入標不可見

現象：dark theme editor 背景為深色，但 CodeMirror caret 未繼承原本 textarea 的 caret-color，且繪製游標的 .cm-cursor 沒有使用 Yuuzu foreground token。

修復：.yz2-cm-host .cm-content 設定 caret-color: var(--yz-e6edf3)；.yz2-cm-host .cm-cursor 設定 border-left-color: var(--yz-e6edf3)。該 token 在 dark theme 是亮色，在 light theme 會覆寫成深色。
