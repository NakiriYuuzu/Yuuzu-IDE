// Everything, one window

A whole workflow, no tab-switching

Yuuzu folds the tools you bounce between — agent, editor, database, terminal, remote and browser — into a single dual-sidebar shell.

Claude Code at the core

Describe intent in plain language. The agent reads files, makes edits with inline diffs, runs your build, and reports back — against your real repo.

Dual sidebar

An icon rail picks a function; the panel beside it shows its contents — explorer, search, source control, databases, remotes and more.

Databases, built in

Browse schemas and run SQL against SQLite, PostgreSQL and MS SQL Server — results render in a grid right next to your code.

SSH + SFTP remotes

Open a shell on production and an SFTP browser side by side. Hosts, keys and files all live in the rail — one click from your editor.

Browser & split view

A built-in browser sits beside your code. Split sends any tab to a second group, so the edit and the result share one frame.

Exportable prompts

Every session captures the structured prompts behind the work. Export them as Markdown, JSON or .prompt to replay a workflow.

// The core

Claude Code is the entry point, not an add-on

A prompt becomes a transcript of real work you can audit line by line — reads, edits with diffs, and shell runs, all inside the editor.

  • Reads before it writes. The agent opens the files it needs and shows you what it found.
  • Edits as reviewable diffs. Every change lands as a +/− diff you can accept or reject.
  • Runs & verifies. Builds, curls and tests run in the same window, with results attached.
