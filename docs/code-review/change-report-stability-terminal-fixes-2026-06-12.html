Why — 為什麼改

診斷報告證實了五個根因：(A) 95/105 個 Tauri command 是同步函式，在 macOS main thread 執行到完成，git 網路、LSP（最長 10 秒）、全 workspace 掃描都會直接凍結整個視窗；(B) terminal 輸出超過 120k 字元後，每個 4KB chunk 觸發 clear() + 全量重寫的 O(n²) 風暴；(C) shell 不是 login shell 且沒設 TERM，brew 指令全部 command not found、打包後無顏色；(D) PTY 固定 80×24 且無 resize 通道；(E) convertEol: true 對真 PTY 輸出是錯的。

實作順序與對應 commit

  • Spec B/P0Spec E/P2562e3fc — Terminal 輸出管線重構（chunk 直寫 xterm）＋移除 convertEol＋Rust 端 10ms chunk 聚合
  • Spec C/P01e3cafe — login shell（-l）＋ TERM/COLORTERM/TERM_PROGRAM/LANG 環境
  • Spec D/P181f381a — PTY resize 通道（後端 command ＋前端 fit/debounce 接線）
  • Spec A/P189983ba — 38 個耗時同步 command 遷移 async/spawn_blocking ＋ LSP 互動 timeout 10s→2s

Architecture — 修正後的輸出管線與執行緒模型

Terminal 輸出管線（Spec B/E）：舊 O(n²) 路徑 → 新直寫路徑

How — 怎麼做的

流程與工具
  • 全程 TDD（依使用者指示無 subagent，所有修改由主迴圈以 Edit/Write 完成）：每個行為變更先寫失敗測試、確認以正確原因失敗、再寫最小實作——replay buffer（7 測試）、terminal-model 形狀（1）、TerminalTab 管線＋resize（8）、AppShell listener（4）、Rust 聚合（4）、login env（3）、PTY resize（2）、LSP timeout（2）。
  • Spec A 的 38 個 command 遷移屬結構性轉換：以 Python 腳本做精確字串替換（每處斷言恰好 1 次出現），行為由既有 324 個 Rust 測試鎖定，逐批 cargo check。
  • 每個 Spec 單獨 commit（規格自訂限制），E 依規格併入 B。
  • 每個 commit 前的閘門：bun test ＋ tsc --noEmit ＋ cargo test ＋ cargo fmt --check ＋ cargo clippy -D warnings 全綠。
關鍵設計決策（含與規格字面的差異）
  • replay 與訂閱的競態：TerminalTab 在 xterm 載入完成後的同一個同步區塊內「write(replay) → subscribe」，而 listener 的「append buffer → publish」也是同步——JS 事件迴圈保證無交錯，所以載入期間到達的 chunk 恰好寫一次（有專屬測試）。
  • SSH 管線一併遷移：Spec B 變更點 3 要求刪除 TerminalTab 的 clear/重寫邏輯，而該元件被 SSH surface 共用，因此 sshOutputBySessionId 系列也移入共用 replay buffer（sessionId namespace 不相撞）。
  • Spec D 與規格字面的差異：規格寫「spawn 時改用 fitAddon.proposeDimensions()」，但 spawn 發生在 TerminalTab 掛載前、當下沒有 fitAddon 可量。等價實作：spawn 仍以 24×80 建 PTY，掛載完成的同一流程立即以 proposeDimensions 實際值呼叫 resize——驗收行為（tput cols 隨視窗、vim 滿版）相同。
  • Spec A 的分層：信任檢查（µs 級記憶體查詢）留在 async fn 本體、await 之前；只有重活進 spawn_blocking。LspState/TaskState 改為 Arc-clonable 讓 closure 取得 'static 擁有權（規格建議的模式）。AppState 的 14 個 lsp 委派方法因此失去生產呼叫者而刪除，其信任測試改測存留的 seam（lsp_workspace_identity、normalize_lsp_document_path、LspState 行為）。
  • 規格的 list_* 保留同步：含 git_list_branches（規格明文「list_*／settings 可保留同步」）。docs_preview/create_text_file 等規格未點名者不動。
  • agent context 的 terminal 快照：改讀 replay buffer 後，輸出 chunk 不再觸發 context useMemo 重算（這本來就是 Spec B 的 render 隔離目標）；attach 時拿到的是最近一次 render 的快照，語意差異極小且方向一致。
  • LSP 互動 timeout 取 2s（規格範圍 1–3s），以 request_timeout_for_method 純函式實作——request 呼叫點不變、可直接單元測試。
Spec C 的根因實證（模擬打包環境）
# 模擬 launchd 基底環境（PATH=/usr/bin:/bin:/usr/sbin:/sbin）
non-login zsh → echo $PATH 含 homebrew 條目數：0（brew 指令全部 not found）
zsh -l → echo $PATH 含 homebrew 條目數：2（/opt/homebrew/bin + sbin）
env -i zsh → TERM=[]（打包後無 TERM ⇒ 無色彩）
spawn env → TERM=xterm-256color + COLORTERM=truecolor（apply_terminal_session_env）

另外確認 SSH terminal 的 request_pty 既有實作已傳 xterm-256color（remote.rs:996），規格「SSH 同步比照」的 TERM 部分原本就滿足。
