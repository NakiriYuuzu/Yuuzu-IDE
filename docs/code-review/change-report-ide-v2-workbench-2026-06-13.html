What & Why（變更摘要）

依照 claude.ai/design 的交付包，把「Yuuzu IDE v2」原型完整實作成 React 元件，成為 app 的預設 shell。設計意圖（出自聊天紀錄）：原本的 IDE「太像 VSCode、功能性不達標」，v2 改為左側專案 rail（多專案、各自保留狀態）＋功能列表（Files / Git / Database / SSH·SFTP / AgentZone）＋型別化 tabs＋AgentZone terminal 集群的全新架構，配 Yuzu 配色（dark/light）。

關鍵設計決策

  • v2 為獨立模組（src/v2/）、v1 完整保留：原 AppShell（9,900 行、接好 180+ Tauri commands 的真實 IDE）一行未動，透過 ?shell=v1 可隨時回去。零迴歸風險。
  • v2 目前使用設計稿自帶的 demo 資料引擎：每個畫面與互動都能在純瀏覽器展示（這正是設計原型的契約）；真實後端（fs / git / pty）的接線是下一階段，逐域把 v1 的 api 層接進 v2。
  • CSS token 與設計稿 1:1 對應：變數名沿用 --yz-<dark-hex>，light theme 用設計稿 helmet 的覆寫值，日後設計回流可以直接 diff。
  • JetBrains Mono 字型隨設計系統帶入（public/fonts/，OFL 授權，6 個字重）。

Architecture（實際結構）

資料流：Workbench 掛載全域鍵盤 → 所有區塊讀寫 v2-store（ui 以 projectId 為 key，切換專案即切換 slice，其餘 slice 原封不動 = 狀態保留）；純邏輯（highlighter、token 估算、欄數斷點、demo 資料）集中在 v2-model 供元件與測試共用。v1 AppShell 完整保留於虛線框。
