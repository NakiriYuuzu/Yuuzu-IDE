What & Why

依指示「直接替代 v1，開始執行」：v2 成為唯一 shell（App.tsx 不再載入 v1；v1 shell 原始碼已自 repo 移除（14 檔，含 9.9k 行 AppShell 與其測試）；原先暫留在 src/app/ 的共用測試與 workspace state 支援已移入非 v1 路徑），並把 v2 接上真實 Tauri 後端 — 完全重用已驗證的 feature api 層，零後端改動。最後依追加需求把 macOS 原生紅綠燈直接內建到 v2 title bar（titleBarStyle: Overlay），消除雙層紅綠燈。

架構決策

  • RealDelegate 注入：store 維持同步、不 import async 程式碼；controller 在 Tauri 環境 bootstrap 時呼叫 registerRealDelegate()，動作在 real 模式分流給 delegate。瀏覽器無 Tauri → 自動 fallback 原本的 demo 引擎（網頁預覽 / 測試完全不受影響）。
  • 純映射層（bridge.ts）可測試：workspace→rail 色組與 glyph、扁平 scan→lazy 樹、GitLogRow→DAG（parent hash→index）、profile/host→側欄卡片，12 個測試覆蓋。
  • 終端機直接重用 v1 戰力：features/terminal/TerminalTab（xterm + fit + replay buffer）原封不動掛進 v2 的 cmd tab 與 AgentZone 視窗；controller 訂閱一次 onTerminalOutput 餵 replay buffer。

Architecture（real ⇄ demo 雙模）

同一套 UI 元件雙模運作：Tauri 內 bootstrapV2 註冊 delegate 並切 mode=real（清空 demo 專案、載入真 registry）；瀏覽器則維持 demo。store 永遠同步，async 全部在 controller。
