  終端機的鐵則 — 在傳統終端機裡,shift+enter 和 enter 送出的位元組完全一樣(都是

  結論:是 ide(yuuzu-ide 終端機)的問題,不是 Claude Code 的 keybind 問題

  你按 shift+enter 之所以直接送出訊息而不是換行,根因在 yuuzu-ide
  的終端機這一層,keybindings.json 怎麼設都救不了。

  根因(三層證據)

  1. xterm.js v6 對 Enter / Shift+Enter 送出完全相同的位元組
  反編譯 node_modules/@xterm/xterm/lib/xterm.js 的 evaluateKeyboardEvent,keyCode
  13(Enter)直接 o.key = CR(\r),沒有 shiftKey 分支。它的 modifier 計算
  (shiftKey?1:0)|... 只套用在方向鍵那種有 CSI ...;mod 模板的鍵上,Enter 不在內。
  2. xterm.js v6 沒有內建增強鍵盤協定
  grep kitty|modifyOtherKeys|CSI u 在整個 xterm lib 只命中 .map source
  map,實際邏輯一個都沒有。也就是說它不會把 shift+enter 編碼成 CSI 13;2u
  這種可區分的序列。
  3. yuuzu-ide 沒有補這個洞
  TerminalTab.tsx:52 建立 Terminal 時只設了 cursorBlink/fontSize/theme,沒有 
  attachCustomKeyEventHandler,onData 把 xterm 產出的位元組原樣轉給
  pty(TerminalTab.tsx:78)。

  → 串起來:你按 shift+enter → xterm 送 \r → pty → Claude Code 收到的跟 enter
  一模一樣 → 觸發 chat:submit 送出。Claude Code
  從頭到尾收不到「shift+enter」這個鍵,所以即使在 keybindings.json 綁
  "shift+enter": "chat:newline" 也是無效的(這也是為什麼 Claude Code 預設換行用
  ctrl+j 而不是 shift+enter)。

  怎麼辦(兩個層級)

  ┌─────────────┬───────────────────────────────────────┬───────────────────┐
  │    方案     │                 做法                  │       成本        │
  ├─────────────┼───────────────────────────────────────┼───────────────────┤
  │             │ 在 yuuzu-ide 終端機裡改用 ctrl+j 換行 │                   │
  │ 馬上能用    │  — 這是 Claude Code 內建 chat:newline │ 零修改,現在就能試 │
  │             │  的預設鍵,ctrl+j = LF\n,xterm         │                   │
  │             │ 會正確傳遞                            │                   │
  ├─────────────┼───────────────────────────────────────┼───────────────────┤
  │ 根治        │ 在 TerminalTab.tsx 加                 │                   │
  │ shift+enter │ attachCustomKeyEventHandler,攔截      │ 改 ~5 行          │
  │             │ shift+enter 手動送 \n                 │                   │
  └─────────────┴───────────────────────────────────────┴───────────────────┘

  根治版的核心邏輯大概長這樣(攔截後送 LF,等同 ctrl+j,Claude Code 就會換行):

  terminal.attachCustomKeyEventHandler((e) => {
    if (e.type === "keydown" && e.key === "Enter" && e.shiftKey) {
      onInputRef.current(sessionId, "\n")  // 送 LF = 換行
      return false                          // 阻止 xterm 預設送 \r
    }
    return true
  })
